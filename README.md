Описание реализованных методов в Heroes-Battle/heroes_task/src/main/java
/programs/ и их сложности.

Метод generate: реализован жадный алгоритм с предварительной встроенной в Java сортировкой алгоритмом TimSort (java.util.List.sort). Сортировка работает за O (n*logn). Жадный алгоритм перебирает типы юнитов за O(n), внутри идёт наполнение армии за O(k), где k – максимальное количество юнитов одного типа. Соответственно жадный алгоритм работает за O(m). Итоговая алгоритмическая сложность: O(nlogn)+O(m)=O(nlogn + m) < O (n*m) (лучше заданной), где n – количество типов юнитов, m – количество юнитов в армии.

Метод simulate: используется встроенная в Java сортировка в каждом раунде и работает за O(nlogn), обработка атакующих юнитов происходит в цикле за O(n), удаление умерших юнитов в конце раунда происходит также в цикле за O(n). Сложность раунда: O(nlogn) + O(n) + O(n) = O(nlogn). Если в каждом раунде в худшем случае сложности умирает один юнит, то сложность всего боя будет O(n^2logn), что удовлетворяет заданной сложности.

Метод getSuitableUnits: реализован алгоритм фильтрации юнитов на базе структуры BitSet для проверки соседних клеток с помощью 2 вложенных циклов. Внешний цикл проходит по всем рядам m раз (O(m)), внутри обрабатывается каждый юнит в цикле, что равно O(n). Итого O(m*n)=O(3*n)=O(n), что удовлетворяет заданную сложность.

Метод getTargetPath: реализован алгоритм поиска в ширину, предсоздание сета препятствий происходит за O(k), где k < WIDTH*HEIGHT. В самом алгоритме поиска каждая клетка посещается не более одного раза O(V), где V < WIDTH*HEIGHT, для каждой клетки проверяются 8 соседей, итого O(8*V). Далее восстанавливается найденный путь в цикле за L шагов, сложность O(L), где L < WIDTH*HEIGHT. Итого: O(k) + O(8*V) + O(L) = O(WIDTH*HEIGHT) + O(WIDTH*HEIGHT) + O(WIDTH*HEIGHT) = O(WIDTH*HEIGHT) < O((WIDTH*HEIGHT)logWIDTH*HEIGHT) (лучше заданной).
